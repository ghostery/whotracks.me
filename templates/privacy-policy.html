Access data / server log files

We log requests to our website via CloudFront's logging mechanism. Each day, a script processes these logs, to derive aggregate statistics on visitors by country from the IP. This script generates a random key for the day, and encrypts all IP addresses with this key. At the same time, we sanitize the HTTP referrers which could contain potentially identifying parameters. The anonymised logs are then copied to a new location, and the key is destroyed once the processing completes. This method allows us to count unique visits from an IP address during a single day, but no day-to-day correlations can be made, nor can the IP address ever be recovered from the anonymised value. The retention period for the raw Cloudfront logs with IP addresses is 24 hours, after which they are permanently deleted.
